Our predefined text and icon classes provide a variety of distinct, actionable button styles.

Button Accessibility

Like our navigation links, buttons include visible hover and focus states to communicate their current state to the user. Icon-only buttons provide text for screen readers.

The Basic Button

All buttons on in the NYC Core Framework require the definer class .btn on an <a> or <button> tag. By default, .btn selectors inherit text color and font size from their parent. The example below is a generic button with no color or style yet applied.

Adding Color

Add color to a .btn selector with our background-color utility classes. These classes automatically apply the button's appropriate text color.

Full-width Buttons

For full-width buttons, add the .display-block display utility class.

Outlined Buttons

For a button with an outline instead of a fill, add the outline modifier class .btn--outline to the .btn selector. The outline width matches your theme's default border width. The outline color inherits the button's text color, which can be changed using our text color utility classes.

Border Radius

Use our border-radius utility classes (e.g. .rounded) for rounded edges on .btn selectors.

Button Sizes

Instead of providing predetermined button sizes (like you'll find in other frameworks) our buttons inherit their font size and use relative (em) sizes for padding, allowing buttons to be resized using our font size utility classes.

This is particularly useful because multiple buttons can be resized within a single selector. In the example below, the small font size utility class (.font-size-sm) is applied to <ul> containing buttons, making all buttons smaller with a single class.

Buttons With Text and Icon

Add an icon to a .btn selector with the .btn--has-icon modifier class. Buttons with icons require two child selectors. The icon itself is a <span> with its combined Font Awesome classes, and an additional class of .btn__icon.

A second <span> with a .btn__text class is needed for the button's text. This selector is our target for underlining the text on the button's :hover and :focus states. It also provides a small margin to separate the text from the icon.

If you wish for a button's text and icon to be opposite of one another, add the chain modifier class --disperse to .btn--has-icon class (resulting in .btn--has-icon--disperse). This class automatically makes the button full-width of its parent.

Icon-only Buttons

Icon-only buttons require the .btn definer class and the .btn--icon-only modifier class. Include the title attribute to describe the button on its hover state (for sighted users).

Like our buttons with text and icon (above), icon-only buttons also require two child selectors. A <span> with an .sr-only class that includes descriptive text for screan readers, and another <span> selector with the combination Font Awesome classes.

Icon-only buttons are transparent by default. Like our other buttons, color can be applied with background-color and text color utility classes.
